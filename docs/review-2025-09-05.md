## Notable Findings (Before Fixes)

- Inconsistent sanitization between JS and native layers:
  - Native (iOS/Android) replaces spaces with hyphens and lowercases; JS service kept spaces and casing.
  - Risk: ch.at TXT routing expects uniform labeling; inconsistent labels may reduce successful resolution and make logs harder to correlate.

- DNS-over-HTTPS (DoH) in JS service was hard-disabled for all servers:
  - Valid architectural point: DoH public resolvers can’t surface ch.at’s custom TXT responses.
  - But for non-ch.at servers, DoH is still useful (web/restricted networks). JS implementation always threw.

- Message validation over-restricted general input:
  - Rejected quotes and other printable ASCII. For the DNS label path, sanitization suffices and native code does not reject common punctuation.

- `test-dns-simple.js` directly requires a TS module:
  - Running `node test-dns-simple.js` fails in plain Node without transpilation. This is a tooling mismatch rather than runtime logic.

## Changes Made

- Align sanitization with native behavior and relax validation:
  - `sanitizeDNSMessage`: now trims, condenses whitespace, replaces spaces with `-`, lowercases, replaces `[.;\\]` with `_`, caps at 63 chars (single-label limit).
  - `validateDNSMessage`: only rejects control characters and empty/whitespace-only input; no longer rejects printable ASCII like quotes.
  - Impact: Cross-platform parity; fewer surprising rejections; better success on ch.at.

- Enable DNS-over-HTTPS for non-ch.at servers in JS layer:
  - `performDNSOverHTTPS`: uses Cloudflare DoH (`/dns-query`) to resolve TXT for non-ch.at targets; still error-fast for `ch.at` with a clear message to use UDP/TCP/native.
  - Improves web support and constrained-network scenarios when using other DNS servers.

- Updated troubleshooting guidance:
  - Clarifies that enabling DoH is effective for non-`ch.at` servers.

- Tests updated and verified:
  - Adjusted `__tests__/dnsService.spec.ts` to match new sanitization (space → hyphen, lowercase).

- Formatting:
  - Ran Prettier repo-wide.

## What I Considered But Did Not Change

- `test-dns-simple.js`: I explored adding `ts-node` auto-registration but modern Expo tsconfig (bundler moduleResolution) conflicts with on-the-fly CJS transpilation. Rather than introduce brittle overrides, I left the file intact and documented the limitation. Two options:
  1) Build step to emit JS for `src/services` and point the smoke test to the output, or
  2) Convert `dnsService` to a small JS shim exporting the methods consumed by the smoke test (retaining TS for the app).

- Excessive verbose logging inside `DNSService` native path:
  - Logs are very detailed, which is useful for debugging but noisy in production. Since logs aid present work, I did not reduce them here.

## Risk Assessment

- Enabling DoH only for non-ch.at targets avoids the known architectural limitation and increases usability elsewhere.
- Validation relaxation avoids spurious rejections while still blocking control chars.
