## Project Overview

React Native mobile app providing ChatGPT-like interface via DNS TXT queries. Features local storage, dark/light themes, and native DNS modules for iOS/Android.

## Core Commands

```bash
# Development
npm start                # Start Expo dev server
npm run ios             # Build and run iOS app in simulator
npm run android         # Build and run Android app in emulator (requires Java 17)
npm run fix-pods        # Clean and reinstall iOS CocoaPods
npm run sync-versions   # Sync versions across platforms
npx expo prebuild       # Regenerate native projects from Expo config

# iOS-specific
cd ios && pod install   # Install iOS dependencies
open ios/DNSChat.xcworkspace  # Open in Xcode (use workspace, not xcodeproj)
xcodebuild -workspace DNSChat.xcworkspace -scheme DNSChat -configuration Debug  # Build from CLI

# Testing
node test-dns.js "message"  # Test DNS functionality
npm test                # Run Jest tests
```

## Architecture

### Tech Stack
- **Framework**: React Native 0.81.4 with Expo SDK 54
- **Language**: TypeScript (strict mode)
- **Navigation**: React Navigation v7
- **Native Modules**: Custom DNS implementations (iOS Swift 5.9, Android Java)
- **iOS Architecture**: Swift-based native module with React Native bridge

### Key Services
- **DNSService**: Multi-method DNS queries with fallback chain
- **StorageService**: AsyncStorage persistence  
- **DNSLogService**: Query logging and debugging
- **ChatContext**: Global state management

### DNS Query Methods (in order)
1. Native DNS modules (iOS/Android optimized)
2. UDP DNS (react-native-udp)
3. TCP DNS (react-native-tcp-socket)
4. DNS-over-HTTPS (Cloudflare)
5. Mock service (development)

## Development Guidelines

### iOS Development
- **Platform Requirements**: iOS 16.0+ deployment target
- **Swift Version**: 5.9
- **Native Module**: `ios/DNSNative/` - DNS resolver using Network.framework
- **Build System**: Xcode with CocoaPods integration
- **Project Files**:
  - `DNSChat.xcodeproj` - Main Xcode project
  - `DNSNative/DNSNative.podspec` - Pod specification for native module
  - `Podfile` - CocoaPods dependencies (auto-generated by Expo)
- **Key Classes**:
  - `DNSResolver.swift` - Native DNS TXT query implementation
  - `AppDelegate.swift` - App lifecycle management
- **Build Configurations**: Debug and Release schemes available

### Android Development  
- **Requires Java 17**: Set via `npm run android`
- Native module in `android/app/src/main/java/com/dnsnative/`
- Uses DnsResolver API (API 29+) with dnsjava fallback

### Version Management
- CHANGELOG.md is source of truth
- Run `npm run sync-versions` before builds
- Updates package.json, app.json, iOS, and Android

## Testing Checklist

3. Verify DNS queries work: `node test-dns.js "test"`
4. Check native module registration
5. Run version sync if needed

## Common Issues & Fixes

### iOS Build Failures
```bash
npm run fix-pods  # Cleans and reinstalls pods
cd ios && rm -rf Pods Podfile.lock && pod install  # Manual pod reset
```

### Native Module Not Found
1. Ensure native module is registered in `ios/DNSNative/DNSNative.m`
2. Check pod installation: `cd ios && pod install`
3. Clean build: Xcode → Product → Clean Build Folder
4. Reset Metro cache: `npx react-native start --reset-cache`

### Xcode Build Issues
- Always open `DNSChat.xcworkspace`, not `DNSChat.xcodeproj`
- Ensure deployment target matches podspec (iOS 16.0+)
- Check Swift version compatibility (5.9)

### Java Version Issues
Use Java 17 for Android builds (automated in npm scripts)
